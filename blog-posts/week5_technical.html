Using Instance Variables in Classes

You've heard this before, but just to hammer it in again, Classes are very important. When programming in Ruby, classes are a major framework of your programming and allow us to program in a truly object oriented way.

So what does that mean? If we want to program certain methods for certain types of objects but not necessarily want them to have the same methods or attributes we create classes. For example, if we wanted to create a class for dogs, we want particular methds for the dog class that wouldn't apply to other objects. An object in the dog class for example might have a method for barking, or a method for fetching a bone. We could also allow each dog to have it's own attributes upon the creation of that instance.

That brings us to instances of a class, what is that? An instance of a class is an object that falls under that class. In our dog example, a dog instance could look like this:

golden_retriever = Dog.new
Our instance, golden_retriever, will now have access to all the methods that reside within the Dog class. But what if we want to provide arguments to this instance? This is where the initialize method comes in.

When we initialize a class like above you can program certain things that will occur when the class is initialized by defining a method within the class as "initialize". The initialize method can accept arguments and then take those arguments to create instance variables like so:
def initialize(name, age, color)
    @name = name
    @age = age
    @color = color
end
Instance variables are very important in classes because they can be accessed throughout the class by different methods. By setting each instance variable like @name to an argument being passed upon initialization we are able to access the data from this instance in all of our dog methods. For example if we have a method in our Dog class that just prints the name of the instance we can use an instance variable to run it.
def name
    print @name
end
Hope this helped your understanding of instance variables, now go include some in your initialize methods!
