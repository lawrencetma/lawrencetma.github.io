We're Uunder Attack!: SQL Injection attacks

With the widespread use of SQL to create and update databases for websites, there were bound to be ways to attack them. SQL injections are how databases are attacked and they can be very dangerous and costly for companies who conduct business online.

So how does SQL injection work? If you can imagine that your code runs queries for the different data fields that are entered into your website, an SQL injection finds weaknesses in your code and exploits it using malicious data inputs that can harm your database.

Some SQL injection can occur when the programmer has not been thurough in making sure that certain data fields are checked to make sure they are a certain type of data such as integers. For example, if a programmer does not check whether a user_id must be an integer the following could be entered as the id:

1;DROP TABLE users
This could be quite problematic as the SQL query will then end up dropping the entire users table like so:
SELECT = FROM userinfo WHERE id=1;DROP TABLE users;
Most SQL injection attacks are a result of programmers who have not correctly written their code to avoid such attacks.

Most of the major ways to avoid SQL injection attacks is to enforce stricter type restrictions on data entries, filtering escape characters such as single quotation marks that can be susceptible to SQL injection attacks.

Because SQL injection attacks are so prevalent and can be used anywhere and almost on every website it is very important that when we create SQL tables for websites that we are following proper coding techniques such as filtering escape characters, checking for data types and for patterns or permissions that might be a weakness in your code.

Make sure you are making the proper precautions when coding your databases in SQL so as to not allow for an easy target of SQL injection attacks. Good Luck and have fun!
